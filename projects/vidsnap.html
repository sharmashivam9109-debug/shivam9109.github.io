Overview

What is VidSnap?

VidSnap is a full-stack video downloader application — not a toy project, but a real production system handling actual users. The backend is a Flask API powered by yt-dlp for multi-platform extraction, with a custom rotation system across YouTube's internal clients (android → mweb → ios → tv → default) to bypass bot detection.

The frontend is a single-file SPA exceeding 5,000 lines — featuring multi-view architecture, quality selection (4K to SD), batch downloads, PWA install, a review section with real social proof elements, bilingual content, and a complete dark theme with glassmorphism aesthetics.

The system runs across two cloud providers simultaneously — Railway and Render — with smart client-side failover logic. Every media URL is served through HMAC-signed proxy endpoints to prevent token leakage and SSRF attacks.

Features

What Makes It Real

🔄
Multi-Client YouTube Rotation
Rotates through android → mweb → ios → tv → default clients to defeat YouTube's bot detection. Falls back to RapidAPI on failure.
🛡️
HMAC-Signed Proxy URLs
Every media URL is signed with a server-side secret and time-limited. SSRF hardening prevents internal network probing.
⚡
Dual-Server Failover
Frontend simultaneously polls Railway and Render. Whichever responds first wins. Cold-start prevention via keep-alive self-pings.
📱
Android PWA
Full PWA with offline support, home screen install, push notifications, and a native app feel — no App Store needed.
🇮🇳
Indian Shorts Support
Dedicated extractor for Indian short-form platforms with Hindi/regional language headers and mobile user-agent spoofing.
🔒
Age-Gate System
Multi-step verification: checkbox → emoji image grid → three confirmation steps. Token generated server-side, cleared on every entry.
📊
IP Rate Limiting
Per-endpoint rate limiting with memory-safe token store. Prevents abuse without blocking legitimate users.
🎯
Quality Selection
From 4K down to SD — auto-selects 720p by default. Format strings cover video+audio merge with ffmpeg fallback.
