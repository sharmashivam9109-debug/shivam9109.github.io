Overview

Native Android Without Native Code

This project packages the VidSnap web application into a native Android APK using Median.co's WebView wrapper platform — but the interesting part is what's injected into the WebView at runtime.

A custom JavaScript bridge is injected into every page load, adding a floating download control panel that communicates with the native layer. The WebView is configured with precise user-agent strings to make the web app believe it's running in a standard mobile browser, while additional native capabilities — push notifications, badge counts, hardware back button handling — are layered on top.

The result is a dual-mode experience: the same codebase runs as both a PWA installed to the home screen and as a native APK distributed via direct download link, with no App Store dependency.

Features

Native-Web Bridge Capabilities

💉
JavaScript Injection
Custom JS injected into WebView on every navigation event — adds a floating download panel, overrides default file download behavior, and intercepts navigation events.
🤖
User-Agent Spoofing
Precise user-agent configuration makes the WebView appear as Chrome on Android to third-party services — avoids platform detection blocks.
🔔
Badge Counts
Download completion events trigger native notification badges on the app icon — bridged from web JavaScript to native Android via Median's API.
↩️
Hardware Back Navigation
Android's hardware back button is intercepted and routed to the SPA router — navigates view history instead of closing the app.
📲
PWA Dual Mode
Web Manifest enables browser-based PWA install. Median.co generates an APK for direct sideload. Same codebase, two distribution channels.
🔒
Secure Storage
Session tokens and preferences stored in encrypted WebView cookies — not accessible to injected scripts from third-party origins.
